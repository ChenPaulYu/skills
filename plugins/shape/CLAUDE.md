# shape ‚Äî plugin-level CLAUDE.md

> Editing this plugin. To run a skill, read its `SKILL.md` (self-contained).

## What it is

**Forward motion: give intent its form, all the way to running, verified code.** Sibling to `nav` ‚Äî the two split the lifecycle cleanly: **shape pushes the work forward** (converge ‚Üí plan ‚Üí build), **nav keeps the result healthy** (audit ‚Üí refactor ‚Üí map). Domain = the loop **converge a decision ‚Üí record it legibly ‚Üí keep it current ‚Üí build it real**.

> Identity note (ADR-011): shape was originally scoped "pre-build". Adding `build` deliberately widened it to forward-motion-through-execution ‚Äî the natural terminus of the spine, a real instance, not scope-creep. nav stays the maintenance half.

## Spine (through-line)

**Converge by a real, disposable instance ‚Äî never a description.** The instance's *form* varies: a verbal grounded fork (elicit), or a rendered interactive artifact (mockup). Its standing form is the blueprints board ‚Äî **one current state, two renders** (agent reads markdown, human reads an HTML projection; deps point downstream only). Its **terminus** is `build` ‚Äî the disposable instances converge the decision; build makes it the durable real thing and verifies it against the mockup (the running system is the third render).

**Progressive disclosure runs through both renders.** Every artifact leads with its interface and drills in only as needed ‚Äî the agent's render of that is markdown structure (a `head -12`-able header + TL;DR, sections that lead with their point), the human's is the click-to-reveal board. Same interface-first principle as `nav`'s file headers; two media.

**The verb is the only door for its deliverable (ADR-038).** If you are about to hand-produce something a verb encodes ‚Äî a comparison mockup (`/shape:mockup`), a converge volley (`/shape:elicit`), a decided behaviour-changing change (`/nav:do`) ‚Äî **fire the verb instead of hand-rolling the artifact**. The artifact is not the deliverable; the *protocol around it* is (post-pick offers, check bracket, gated diffs) ‚Äî hand-rolling produces the artifact and silently drops the protocol. Having done the craft ambiently many times is the *risk signal*, not a waiver: fluency is exactly when the door gets skipped (two sightings: 2026-06-03 execution, 2026-06-11 converge). This does not make verbs auto-fire ‚Äî it governs the case where you're *already producing* the deliverable.

**Canon is single-writer and freeze-gated (ADR-041).** A project's core docs (`docs/core/*`-class ‚Äî its constitution) are written by exactly one door: an explicitly-summoned `/shape:position`, and even then only at freeze moments (campaign close ¬∑ explicit per-item freeze order ¬∑ root-doc birth) ‚Äî mid-campaign rulings land in a campaign log, because **ruling ‚âÝ ratification**. Every other verb and all ambient conversation: never write core ‚Äî append a one-line pending amendment to `docs/core/amendments.md` (what changed ¬∑ evidence ¬∑ by which verb); position adjudicates the ledger as each summon's first feeding, and reconcile sweeps its leftovers. Freshness-law carve-out: stale header = fix same-commit; **stale canon = ledger entry (tracked debt), never a direct edit**. This is ADR-038's clause at the canon layer. Full protocol: `plugins/shape/skills/position/references/write-protocol.md`.

**Below 90% ‚Üí ask (core principle, shared with `nav` rule ‚ë¶).** When unsure about scope, boundaries, or intent, stop and ask ‚Äî never guess. This is the spine of the whole family, not a per-skill add-on: `elicit` *is* this rule as a workflow (it converges by asking, not answering); `mockup` lets the user point; `align` decides *with* the user; `reconcile` gates every destructive step per-file. Any autonomous driver built on top runs **only while confidence holds**. The moment scope/boundary/intent drops below 90%, it stops and asks rather than plowing ahead ‚Äî the cost of pausing to clarify is far smaller than unwinding a wrong guess.

## Members

Grouped by verb (mirrors `nav`'s family shape):

- **converge** ‚Äî turn an open question into a decision:
  - `mockup` ‚Äî render-to-decide: a real interactive HTML (UI mockup, or diagram for backend/agent/data/flow). *(built)*
  - `elicit` ‚Äî draw the decision out by a grounded fork (react-not-author, drill-to-principle, compress-to-one-line; weight-adaptive exit ‚Äî stop on the snap; summoned, not auto) ‚Üí a `thoughts/` doc. *(built)*
  - `dogfood` ‚Äî use a *built* feature that feels unsmooth for real: drive the REAL interface (`agent-browser` / `curl` / CLI) against a list of user intents, report **friction** (works but clunky ‚Üí a UX idea) + the **coverage gaps** that fall out (an intent with no path ‚Üí a logic hole). Experience-first; logic-coverage is the byproduct. Output is a friction report (not a mandatory mockup); render is an optional redesign hand-off. A gap's layer (missing intent = direction ¬∑ dead-end = incomplete) routes it, handed to `elicit`. Borrows `/verify`¬∑`/run`'s drive-the-real-app method (asks "is it smooth + what's missing", not "is it correct"). *(built; ADR-033, supersedes ADR-020)*
  - `position` ‚Äî **author the canon layer**: a gated multi-feeding *campaign* that lands core (principle-wise) docs. Five kernels: **ingest-assess** (per-feeding delta report ‚Äî new / metabolized / conflict / needs-ruling ‚Äî with *guarding*: conflicts with locked decisions block, not absorb) ¬∑ **altitude instrument** (axiom/principle/approach/bet + the *churn alarm*: a second flip on the same decision = form-layer debate, lift to the rule) ¬∑ **two-tier landing** (`core/` = user-ratified + self-contained; `thoughts/` = dated hypotheses) ¬∑ **graduation** (core *grows* by user-frozen thoughts, never scaffolded) ¬∑ **periodic re-audit** (canon accretes mid-campaign; pre-landing `think:first-principles` self-audit). **Doc-class framed** ‚Äî any core doc, not just product positioning. **Write protocol (ADR-041)**: core single-writer + freeze-gated ‚Äî mid-campaign rulings ‚Üí the campaign log; other verbs ‚Üí the `docs/core/amendments.md` ledger, adjudicated as each summon's first feeding. Mirror of `reconcile`: position *births* canon (pre-code, authority-driven), reconcile *keeps it true* (post-code, reality-driven). A meta-skill (ADR-008): borrows elicit / mockup / first-principles by protocol. *(built; ADR-035, write protocol ADR-041)*
- **project** ‚Äî render the current plan:
  - `align` ‚Äî read `thoughts/` + real state ‚Üí decide now/next/later *with the user* ‚Üí write `plan.md` (agent) + regenerate `overview.html` (human). The pre-build mirror of `/nav:sync`'s codebase map. *(built)*
- **reconcile** ‚Äî keep the archive honest:
  - `reconcile` ‚Äî judge which `thoughts/`, `plans/` **and `mockups/`** entries have drifted (code/headers ¬∑ self-declaration ¬∑ date) ‚Üí **amend** a stale *fact* in place, **prune/consolidate** a wholly-stale doc, or **retire** a shipped decision's mockup folder (salvage residue ‚Üí prune, git as deep archive; parked decisions + stamped structural locks stay), *with the user*, write-gated. Amend syncs only what built code shows; a *decision*-change is out of scope ‚Üí it recommends `/shape:elicit` and says so (the active form of "no new decisions during render"). Also sweeps the **`docs/core/amendments.md` ledger** (absorbed/reversed entries ‚Üí prune ‚Äî the ledger's exit) and flags **canon-grade** thoughts (settled axiom/principle ‚Üí recommend `/shape:position`, never write core; ADR-041). The pre-build mirror of `/nav:audit` + careful `/nav:refactor`. *(built; amend in ADR-014; maintains `plans/` per ADR-017; sweeps `mockups/` per ADR-037 ‚Äî the enforcement point of mockup's retire-on-ship; ledger + canon-grade per ADR-041)*

## The seams with `nav` (hard deps one-way; soft recommendations either way)

shape is the forward-motion half; `nav` is the maintenance half. The dependency rule has two tiers:
- **Hard deps (import / call / breaks-without-it) are one-way: shape ‚Üí nav.** `nav` must run fully standalone ‚Äî never *depend* on shape.
- **Soft recommendations (a guarded *offer* / hand-off that degrades if absent) may go nav ‚Üí shape.** These already exist and are fine: `/nav:plan` Stage 2 hands visual ambiguity to `/shape:mockup`, and Stage 4 *offers* a `/shape:mockup` summary (guarded ‚Äî omitted if shape isn't installed). See ADR-012.

Record the seams; don't blur them:

1. **`blueprints/` is the hand-off artifact to `/nav:plan` ‚Äî both directions.** shape converges intent into blueprints; `nav:plan` grounds a thought/spec into a code-level implementation plan **and writes it back into `blueprints/plans/`** when the tree is present (soft `nav ‚Üí shape` preference, ADR-017). `align` triages forward; `nav:plan` grounds one item down ‚Äî adjacent verbs, not overlapping. The grounded plan stays co-located so the whole arc (decision ‚Üí status ‚Üí grounded-how) lives in one tree.
2. **`reconcile` consumes the file headers `/nav:sync` maintains, and maintains `plans/` + `mockups/`.** `head -12` file headers make "is this implemented?" answerable cheaply ‚Äî the strongest staleness signal. reconcile keeps **`thoughts/`, `plans/`, and `mockups/`** current (a plan whose steps all shipped is stale like an implemented thought; a shipped decision's mockup retires once its residue is absorbed ‚Äî ADR-037).
3. **`build` is the concentrated seam.** It *controls* the loop (lives in shape, forward-motion) while *calling* nav's code-side protocols per item: `/nav:plan` (ground) ‚Üí `/nav:refactor` discipline (implement) ‚Üí `/nav:sync` (cheap header grounding for inject), then writes back via `/shape:align`. Direction stays one-way: shape ‚Üí nav.

## Conventions for skills inside this plugin

> Repo-wide **authoring + maintenance** rules (naming, skills-root-relative paths, stack-neutral examples, frontmatter `description`, ADR-on-new-skill, the site-map gate, versioning) live in the repo-root [`CLAUDE.md`](CLAUDE.md). Shape-specific:

- **‚òÖ Progressive disclosure in every produced artifact** (so an agent scans fast, not just the human): a `thoughts/` doc opens with `# <title> ‚Äî <one-line role>` + a ‚â§3-line TL;DR/status blockquote (`head -12` yields the gist without reading the body), sections lead with their point, enumerations go in tables, key terms bolded; a `mockup` artifact opens with a top-comment stating what it is ¬∑ the candidates ¬∑ the pick. This is `nav`'s interface-first (rule ‚ë°) applied to shape's outputs. Every skill that emits a doc/artifact (`elicit`, `mockup`, `align`) enforces it.
- **Write-gated**: `align` and `reconcile` write files ‚Äî show what will be written (or a diff) before committing. `reconcile`'s destructive ops follow the safety rules in its `SKILL.md` (check tracked/untracked, never `mv`-then-`rm`, diff before delete).
- **Skills don't invoke each other**: they reference sibling protocols by name (e.g. reconcile ‚Üí "run `/shape:align`"), never re-implement or call them.
- **Cost tier (ADR-058/059)**: the *criterion* (mechanical sweep/format/render ‚Üí `model: sonnet` frontmatter, turn-level) is owned by the repo-root [`CLAUDE.md`](CLAUDE.md); shape's tiered verbs are **`reconcile`** and **`align`**. Judgment-heavy verbs (`elicit`, `position`, `build`'s adjudication) stay on the session model. The same tier powers the browser-verify slot's `browser-verifier` subagent (below).

## browser-verify ‚Äî a shared per-project capability slot

Three skills need to *see* the running thing ‚Äî `mockup` (render the artifact), `align` (open the board), `build` (verify each item against its mockup). They share **one** pluggable capability, not three copies (N+1 trigger):

- **Default implementation: `agent-browser`** (vercel-labs/agent-browser) ‚Äî a browser-automation CLI. Detect: `which agent-browser`. Usage: `agent-browser open <url>` ‚Üí `agent-browser screenshot <file>.png`; `snapshot -i` / `click @e` / `fill @e "..."` to interact.
- **Don't hardcode the tool ‚Äî name the capability.** A skill describes the contract ("open the running feature ‚Üí screenshot ‚Üí compare to mockup ‚Üí **close when done**"); `agent-browser` is the named **default**, not the contract. The close is *part of* the contract, not optional cleanup: an unclosed headless browser silently burns a core after the pass returns, and the cost lands on the user's machine ‚Äî invisible from inside the loop, so it has to be in the contract, not left to judgment (trap #3). Caveat: this written contract is *polled* knowledge, not a *cue* ‚Äî it fires only if the executing path actually walks it. So bind the close where open *executes*: write each verify step so its last move is `close` (a walked step is a cue, a reference doc is not), and lean on a turn-end browser reaper as the cross-path backstop. The doc raises the odds it's in-context; it does not make it fire.
- **If missing, fail helpfully ‚Äî never silently skip.** Surface a 3-way confidence-gate choice: **(a) install** [recommended] ‚Äî `npm install -g agent-browser` (or `brew`/`cargo install agent-browser`) then `agent-browser install`; or as a skill `npx skills add vercel-labs/agent-browser`. **(b)** proceed without visual verify this run (flag items to eyeball). **(c)** skip verify for this item. Report what was skipped (no silent caps).
- **Per-project override:** a project may bind a different helper (Playwright, puppeteer) in its own CLAUDE.md; the agent reads that. Absent an override, the default is agent-browser.
- **Skills don't call skills:** name the capability + describe the contract; the executing agent invokes the tool (CLI shell-out, or triggers the agent-browser skill in its own turn).
- **Delegate the pass to the `browser-verifier` subagent ‚Äî the cost tier (ADR-058).** The slot's default *executor* is the plugin's [`agents/browser-verifier.md`](plugins/shape/agents/browser-verifier.md) (model: sonnet). The caller injects URL ¬∑ screenshot destination ¬∑ expectation (mockup path or stated behaviour) ¬∑ any interaction steps / tool override, and receives back a compact verdict (`PASS | DRIFT | BLOCKED | MISSING-TOOL` + reason + paths) ‚Äî the drive-and-capture loop and its **image tokens stay in the subagent's context**, never entering the main conversation (where every screenshot is re-paid on every later turn). Delegate whenever the pass is mechanical (build's per-item verify, mockup's capture confirmation, align's board check); drive inline only when the user is being walked through the page live. The close-contract binds inside the subagent ‚Äî its last move is `close`. Ships with the plugin, so it works on any machine that installed shape.
- **Verify economy ‚Äî capture at decision points, not per step.** A screenshot is *evidence* (Shipped-evidence, friction-evidence, a pick's record), never a progress note: `build` captures once per item at land (mid-item verification is the test gate); `dogfood` screenshots at friction/gap moments and records video **only on explicit request**; `mockup` confirms a render once, not per iteration; `align` opens the board without capturing. Screenshots live on disk (report/mockups tree) ‚Äî referenced by path, not pasted into the chat.
- **Known traps:** the slot's field manual ([`references/browser-verify-gotchas.md`](plugins/shape/references/browser-verify-gotchas.md)) is organized by scope ‚Äî **universal** (session isolation ¬∑ assert state-not-text ¬∑ measure-before-toggle ¬∑ selector-over-pixel clicks ¬∑ poll-don't-snapshot ¬∑ rule-out-mock-data ¬∑ daemon-reset ¬∑ teardown/`close` with its knowledge‚âÝtrigger cue-ladder + provenance-keyed reaper), **audio** (dual-backend mute ¬∑ real-input for user-activation), **transform-canvas** (`position:fixed` ‚Üí portal to body), and **React+Vite** (controlled-input native-setter, HMR console ghosts). Read before trusting a screenshot or console scan, and `close` the browser when the pass ends.
