# shape — plugin-level CLAUDE.md

> Editing this plugin. To run a skill, read its `SKILL.md` (self-contained).

## What it is

Pre-build: **give intent its form before building.** Sibling to `nav` (post-build code navigability).
Domain = the pre-build pipeline **thought → spec → plan**.

## Spine (through-line)

**Converge by a real, disposable instance — never a description.** The instance's *form* varies by stage: a verbal grounded fork, or a rendered interactive artifact.

## Members

- **Skills** (callable, each an action):
  - `thought-ground` — summoned grill: converge a conceptual decision by a grounded fork. *(unbuilt)*
  - `mockup` — render-to-decide: converge a visual / structural decision by a real interactive HTML (mockup or diagram). *(built, v0.0.1)*
- **`spec` = convention**, not a skill — the WHAT artifact contract (spec = interface / plan = implementation / progress = state; one-way deps).
- **`plan` = reuse**, not a skill — `planning-with-files` (progress.md, cross-session) + the layered model.

## Status

Early — only `mockup` built. The family crystallizes (ADR, full structure) as skills prove out. History: `docs/observations/2026-05-29-shape-plugin.md`.
