# First-principles positioning check: ask what made the comparison level what it is

> Source: Enact paper research session — user said "I want to position this at the same level as ReAct." The right response was to check this from first principles rather than affirm it, which led to a more honest and defensible positioning.

## The case

User: "I want to position Enact at the same level as ReAct — a new agent runtime paradigm."

Naive response: affirm the positioning and build from there.

Better response: check from first principles — what made ReAct paradigm-level? Does Enact have the same properties?

**What made ReAct paradigm-level:**
- Solved a domain-agnostic problem (reasoning+acting for any task)
- Its loop applies to any agent without requiring a pre-defined identity
- Changed the entire field's thinking
- Required no prior knowledge about the agent's "character"

**What Enact actually is:**
- Requires a pre-defined identity to run
- Only meaningful for character-based agents
- Will be adopted in that subfield, not universally

Result: Enact is framework-level (like Generative Agents), not paradigm-level (like ReAct). This is a more honest and ultimately more defensible claim.

The user responded: "I agree, but we can gesture toward generality as an open question." → cleaner paper with honest scope + forward-looking Discussion section.

## The move

When a user wants to position their work at level X, check:
1. What made X that level? (list 2-3 specific properties)
2. Does the new work have those same properties? (honest check)
3. If not: what level IS it actually? Find the right analogy (e.g., Generative Agents rather than ReAct)
4. Offer the honest framing — it's almost always stronger than the inflated one

This is not about deflating ambition. Generative Agents is not "worse" than ReAct — it's a different kind of contribution, and both are valuable. The honest framing protects against reviewer attacks.

## The failure mode

Affirming the inflated positioning without checking. This leads to:
- Reviewers pointing out the scope mismatch
- Evaluation designed for the wrong level (trying to show universal generalization when the claim is domain-specific)
- Related work that competes with the wrong papers

## The residue from this case

The right question that unlocked better positioning: "From first principles, does this have the same properties that made X paradigm-level?"

Often the honest answer is "no, but it's still valuable as Y" — and Y is usually a cleaner, more defensible paper.

## Connection to research skill

This check belongs at the start of a research skill's "positioning" step — before finalizing what the paper claims, ask the first-principles level check. The output is a positioning statement with honest scope + a "toward generality" gesture if appropriate.
