# Drilling to the atom — two failure modes: surface-conflation, and over-rotation when pierced

> Complements [[2026-05-29-thought-mode-how-paul-converges]] + [[2026-05-29-how-diagnosis-converges]], which capture the convergence **method** (`shape:elicit`'s "drill to the most fundamental layer · react-not-author · the user pierces · reframe"). This is the **failure side** — two ways an agent botches the drill. Source: one Crate session designing step-2 AI integration, where both fired repeatedly and the user's "用第一性原理 / 真的是這樣嗎?" pressure-testing surfaced them. **Not promoted**: `raw`, one session — a candidate Discipline / anti-pattern pair for `elicit` if it recurs.

## The method these break (context)

`elicit` says: when the user pressure-tests, go *under* the surface question to the principle beneath; converge to a principle, not a feature list; watch for the reframe. The skill describes the *target* (the atom). Two failure modes block reaching it.

## Failure mode 1 — conflating surface-similarity with sameness

The agent calls two things "the same" because they share a *surface* feature, missing that the deciding unit is a layer down.

Real (Crate AI session):
- **"Chat and deep-research both type text into a box → same input, so one composer."** Wrong: the atom isn't "typed text", it's the **speech act** — a *dialogue turn* (elliptical, "那他呢?", needs you in the loop) ≠ a *self-contained commission* (a brief you hand off). Once "speech act, not typed text" was named, the whole "same place?" question dissolved.
- **"Adding context and displaying context both involve context → the in-card list duplicates the wire."** Wrong: a per-edge *working wire* (spatial, "that card feeds this") and an aggregate *roster* (complete-set-at-a-glance) do different jobs; "both involve context" is a surface match.

**Tell:** the word "both …" doing load-bearing work in an argument for sameness / redundancy. Ask **"same atom, or just surface-similar?"** before collapsing them.

## Failure mode 2 — over-rotation when pierced

When the user refutes a position, the agent swings to the *opposite* extreme instead of going *down* a layer.

Real: pushed on "deep research = a composer gear", the agent over-corrected to "it's not a gear at all — it's a right-click action on an object" — overshooting. The user pulled it back ("but it starts from a single question"). The right answer was neither pole but a layer down: deep research is a **commission** (shares the question-entry with chat, diverges only in execution) — which appears only once you stop defending *and* stop swinging, and locate the atom.

**Tell:** a refutation triggers a same-turn flip to the contrary position. The pierce is a signal to **descend**, not to **invert**.

## The principle

> **When pierced, go DOWN a layer (name the irreducible atom) — don't go SIDEWAYS (flip to the opposite). And before calling two things "the same", check they share an *atom*, not just a *surface*.**

Owning an over-rotation out loud ("I over-rotated, here's the sharper version") keeps the user building with you instead of re-litigating.

## Why it's outward-useful (and why raw)

Not person-specific — any first-principles convergence (`elicit`, its diagnosis variant) is vulnerable to both, and an LLM is *especially* prone (surface tokens are cheap to pattern-match; a refutation easily reads as "pick the other option"). If they recur across projects, they're a Discipline / anti-pattern pair on `elicit`'s SKILL ("don't conflate surface-similarity → check the atom; when pierced, descend not invert"). One session so far → `raw` (ADR-018 gate).
