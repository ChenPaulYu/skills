# ADR 080 — Generalize `research`'s object from "paper" to "argument document"

**Status**: accepted
**Date**: 2026-07-14
**Source**: ratified by Paul 2026-07-14, via `/shape:elicit` — [`blueprints/thoughts/2026-07-14-research-generalize-argument-docs.md`](../../blueprints/thoughts/2026-07-14-research-generalize-argument-docs.md).
**Precedent cited**: [ADR-027](027-research-plugin.md) (research plugin charter — already named the framework domain-agnostic at birth); [ADR-012](012-nav-plan-offers-visual-summary.md)-pattern guarded offers (the seam convention this ADR instantiates three times).

## Context

`research` is the marketplace's only family organized around an academic object. Every other family names a domain-neutral verb-space: `nav` keeps *codebases* navigable, `shape` pushes *work* forward, `frame` applies a *lens* to a problem, `reflect` looks at a *session*, `relay` coordinates *people*. `research`'s own `CLAUDE.md` (pre-this-ADR) opened with "a collection of skills for reading external sources with intent," but its worked examples, its `notes/<paper-shortname>.md` save-path convention, and its frontmatter descriptions ("review this paper," "adversarially review a paper") all centered the *paper* as the unit — an academic artifact — even though the underlying framework never required it.

This produced a real isolation symptom: `research` is the only plugin with **zero recorded seams** to `shape` or `frame` in `plugins/research/CLAUDE.md`, despite three natural hand-offs sitting unrecorded in already-shipped behavior — `untangle`'s positioning view is exactly canon-shaped once a claim is ratified (a `/shape:position` job); a critique finding that says "no evidence exists either way" (not "the evidence shown doesn't support the claim") is exactly a `/shape:probe` job; and `/shape:survey`'s existing `deep-research` offer for external-knowledge gaps was never connected back to `research`'s own domain in any doc. The family read as a domain island not because its skills lacked substance, but because its own identity prose never stated where its outputs go next.

### Diagnosis: two candidate root causes

1. **(A) Object-definition error** — "paper" was mistaken for the family's defining unit, when it's actually one instance of a broader class: any document carrying a claim-evidence structure (a paper, an RFC, a design proposal, an ADR, a whitepaper, a technical blog post). Paul's framing: *a paper is one kind of document in the world, not a unique element.*
2. **(B) Missing downstream seams** — the family works standalone and never hands its outputs to `shape`'s canon-authoring or experiment-running verbs, even though `dissect`'s own description already listed RFCs and design proposals as valid input (evidence the object was already half-general in practice, just not in the family's *self-description*).

**Paul ruled A is the root cause; B is a real, independent gap to fix in the same pass — not a competing diagnosis.** Fixing B without A leaves the family correctly wired to `shape` but still describing itself as a paper-reading tool, which would just relocate the misdescription. Fixing A without B leaves the object correctly named but no less isolated. Both land here.

### Two ways to fix the object error, and why not the cheaper-looking one

- **A1 — scatter the four verbs into `nav`/`frame`.** `dissect`/`critique`/`provenance`/`untangle` all produce artifacts (dissection notes, referee reports, audit notes, relational maps) through multi-step, sub-agent-dispatching **campaigns**. `frame`'s three-house rule (`plugins/frame/CLAUDE.md`) is explicit: its lenses run **in-chat, write no file, one cognitive move per invocation** — `first-principles` derives an answer, `dialectic` puts a claim on trial, `orthogonal` factors a tangle, each closing in a single turn with an optional route-out offer. None of research's four verbs fit that shape: `dissect` alone dispatches a sub-agent per document and writes a saved note; `critique` runs a six-step protocol landing two artifacts plus a cut-log. Moving them into `frame` would break frame's own identity, not extend it. `nav` fares no better — its object is *code*, and grep/header/refactor tooling doesn't transfer to "read this RFC and audit its citations." A1 fails the protocol-shape law before it clears the first plugin boundary.
- **A2 (this ADR) — generalize in place.** Keep all four skills where they are, in the plugin they've always lived in; rewrite the family's own self-description so the object it names (argument documents) matches what the skills already do. No file moves, no engine changes, no protocol rewrites.

A2 wins by elimination: A1 is foreclosed by frame's protocol-shape boundary, and there is no serious third option that doesn't also either violate a sibling family's shape or duplicate machinery that already exists in `research`.

### Rename vs keep-name

A second fork, orthogonal to A1/A2: should the plugin itself be renamed away from "research" (which carries an academic connotation in casual English) to something more neutral?

**Ratified: keep the name `research`.** Two reasons:
1. **Namespace migration is this repo's heaviest single operation.** Every `/research:*` invocation, every cross-reference in every other plugin's `SKILL.md`, `README.md`'s tables, `docs/site/index.html`'s CB_NODES/DOMAINS/CONV entries (both languages), `platforms/codex/descriptions.json`, and the generated `.agents/skills/` mirror would all need coordinated rewrites — for a cosmetic gain, not a functional one. ADR-078's relay-register merge and ADR-079's summarize retirement both show how much surface area even a *smaller* structural change touches; a rename touches strictly more (every invocation site, not just the roster).
2. **"Research" in plain English already means rigorous inquiry, not academia.** The dictionary sense — "diligent and systematic inquiry into a subject to establish facts" — never required a university affiliation or a peer-reviewed venue. The plugin's own identity line ("rigorous reading and auditing of argument documents") is a restatement of that ordinary meaning, not a redefinition. The isolation this ADR fixes was never really about the *word* "research" — it was about the family quietly narrowing that word's natural scope down to "papers" in its examples and defaults. Un-narrowing the scope resolves the actual problem; renaming would not have, and would have cost far more.

## Decision

1. **`plugins/research/CLAUDE.md`** — identity paragraph rewritten: the object is **argument documents** (papers, RFCs, design proposals, ADRs, whitepapers, technical blog posts — anything with a claim-evidence structure); "paper" demotes explicitly to "the most common instance, not the definition" (the car/vehicle analogy is stated inline). Roster line for all four skills updated to say "documents" not "papers" where the swap costs nothing. A new **seams** paragraph records the three hand-offs named below.
2. **Four `SKILL.md` frontmatter descriptions** — each keeps its existing paper-example trigger phrase (papers are a valid, common instance — not banned) and gains **one additional non-academic trigger phrase**:
   - `dissect`: adds "break down this RFC"
   - `critique`: adds "stress-test this design doc's claims"
   - `provenance`: adds "audit my ADR's citations"
   - `untangle`: adds "untangle these design proposals"
   Every existing NOT/boundary sentence is preserved verbatim — these are load-bearing for skill routing (the family's own routing probe already validated 51/51 against the current boundary wording; a generalization pass that touched those sentences would need to re-run that probe).
3. **Body sweeps, not restructures** — each `SKILL.md` swept for "the paper" used as if it were the only possible input; generalized to "the document" where the swap changes nothing else. Save-path conventions (`notes/<paper-shortname>.md`) generalized to `<doc-shortname>.md`. Worked examples that are papers are kept (they're valid instances); each skill gains at least one explicit non-academic example in its Scope section (critique: RFC / design proposal / vendor whitepaper degrade-gracefully note). **Protocols and engines are untouched** — this is narrative convergence (the prose matching what the skill already does), not a behavior change. No version-bump-worthy logic changed; the bump below is for the roster-adjacent identity rewrite, consistent with how this repo has bumped past minor-version identity edits.
4. **Three seams, recorded (not built — they already existed as latent capability):**
   - **`untangle` → `/shape:position` / `/shape:elicit`.** The positioning view (Step 4, when a claim is given) is stated explicitly as input to both: `/shape:position` when it's ready to ratify as canon, `/shape:elicit` when it still needs converging. Guarded offer, ADR-012 pattern, stated in untangle's own Output section + a new Companion skills section.
   - **`critique` → `/shape:probe`.** Mirrors `/frame:dialectic`'s existing `/shape:probe` bridge: critique audits *existing* evidence; when a Weakness is really "the decisive experiment doesn't exist yet," `probe` is where it can go to actually get run. New Companion skills section, same guarded wording as dialectic's.
   - **`/shape:survey` → `deep-research`.** Already existed (survey's Step 3 "big gap" fill offers `deep-research` for external-knowledge gaps). This ADR records it as `research`'s ecosystem-adjacent counterpart — survey's offer is the terrain-mapping sibling to research's document-reading verbs, both eventually routing to the same external-research capability for what's outside the documents already on hand.
5. **Deep-research cost label (Paul-ratified option b, same day).** Both places that offer `deep-research` — `/shape:survey`'s big-gap fill and `/research:provenance`'s new quarantine remediation for a true orphan (no known candidate source at all) — now append one sentence, in the offer's own wording, naming the cost: deep-research is a multi-agent research harness that fans out dozens of searches and verification rounds, an order of magnitude above the sibling remediations/offers next to it. This is an [ADR-062](062-live-llm-cost-signal.md) instance: aware-while-writing, notify-at-the-offer, never an automatic swap to something cheaper — the user still picks, now with the cost named instead of implied.
6. **Capability first — artifact folders demoted from quasi-contract to convention (Paul-ratified, same day).** The family's fixed folder bindings are dissolved: the **in-chat analysis is the deliverable** (skeleton / referee report / relational map); a saved file is an end-of-run *offer*, never a requirement, and declining to save is a valid outcome. `notes/` (analysis notes) and `sources/` (fetched originals, e.g. a URL downloaded during intake — newly recorded, previously implicit) become **suggested defaults** under [ADR-071](071-contracts-vs-conventions-tolerant-reader.md)'s contracts-vs-conventions law — any user-named location works, and location never affects the analysis. Cross-verb reuse (untangle/critique finding a prior dissect note), previously a hardcoded `notes/<shortname>.md` lookup, becomes a **tolerant three-state lookup** with self-report: ① this conversation → use directly; ② user-pointed location → read there; ③ conventional `notes/` → else derive from the document itself. The honest trade this accepts: reuse degrades from "guaranteed hit when the folder is shaped right" to "best-effort" — which is exactly what ADR-071 says a convention should be; the capability (the analysis) no longer has a hard dependency on any folder shape.

## What does NOT change

- **Engines and protocols** — dissect's inject→read→check sub-agent dispatch, critique's ground→self-attack coupled core, provenance's three-verdict trace, untangle's four-edge relational structure: none of these change shape, step count, or discipline. This ADR is prose, not behavior.
- **Routing boundaries vs `/nav:compose` and `/frame:dialectic`** — `research` still only *reads*; authoring a new document is `/nav:compose`'s job, not research's (research's write surface stays "a fixed-format note about what was read," never original argument-authoring). `/frame:dialectic` still owns stress-testing *your own forming claim* where the decider doesn't exist yet — `critique` still only audits an *external* document's *existing* evidence. Both boundary sentences are preserved verbatim in the relevant `SKILL.md` frontmatter (point 2 above); this ADR adds a bridge to `/shape:probe` alongside the boundary, it does not blur it.
- **Plugin name, dependency posture** — `research` stays independent (no `nav`/`shape` hard dependency); the three seams above are soft, guarded offers, degrading gracefully if `shape` isn't installed, same as every other cross-plugin offer in this repo.
