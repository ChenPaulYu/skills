# ADR 070 — `reflect:park`: the write side of the cursor (reflect becomes two writers)

**Status**: accepted
**Date**: 2026-07-13
**Source**: `blueprints/thoughts/2026-07-13-reflect-park-writes-the-handoff.md` (gap statement + two ✅ decisions); `docs/findings/2026-07-13-park-ab-experiment.md` (the empirical test of the proposal).

## Context

Of reflect's three existing members, `catchup` reads, `summarize` reads, and only `observe` writes — and `observe` writes a distilled *learning*, not "where the session is and why." `catchup`'s own law is "grounding is the value, not chat paraphrase" — correct, but durable state has never had a *why* in it for `catchup` to ground on, because no verb ever wrote one there. `catchup` can only reconstruct "roughly where things are" from git/diff/on-disk layout; it cannot recover "why this was chosen" if that reasoning only ever lived in a conversation that later disappeared.

**Three independent sources** converge on the same structural gap:

1. **softaworks/agent-toolkit's `session-handoff` skill** — a fixed 10-section template stored as dated files (`.claude/handoffs/YYYY-MM-DD-HHMMSS-slug.md`), chained via `--continues-from`, with a **staleness check on resume** (compare recorded state against current git). Explicitly **not** adopted here: its 80%-context auto-trigger.
2. **A 2026-03 context-degradation study** (cited secondhand, original not independently verified) — reasoning quality degrades starting at 20–40% context usage, well before the window is full, with a "rushed-finish" pattern appearing near the limit.
3. **mattpocock/skills' `ask-matt` skill** — names ~120k tokens as the reasoning "smart zone" boundary and recommends a `/handoff` before crossing it, rather than running until the window is full.

Per [ADR-018](docs/adr/018-promotion-gate-is-evidence-not-session-count.md)'s recurrence-path standard (≥2 independent sightings clears the bar, no need to wait for 5), this — plus reflect's own internal gap (only `observe` writes, and not at cursor granularity) — is sufficient grounds to scaffold.

**Empirical test, not just citation.** Before shaping the skill, the proposal was tested: 6 mutually-blind agents were handed one of three arms (bare git / an unstructured notes file / a `HANDOFF.md`-shaped handoff with five sections + git SHA + explicit dead-end markers) against two synthetic mid-work repos, each seeded with a "red herring" — a plausible next-step that had actually already been rejected in a prior session. Results: bare git scored 11/20 on the five-question rubric and produced both hallucinations recorded across the whole test — critically, **it didn't just omit the missing why, it confidently invented one** (a rejected performance theory, stated as fact, with no basis in the code). An unstructured notes file recovered most of the gap (19/20) — "having written *something*" carried most of the value. The structured handoff also scored 19/20 but was the only arm with **zero hallucinations and a clean pass on both red herrings** — its explicit "already rejected, don't retry" line is what neutralized the trap; the unstructured notes arm's vaguer phrasing on the same point left it ambiguous enough to produce that arm's one hallucination. Full method and numbers: `docs/findings/2026-07-13-park-ab-experiment.md`.

## Decision

1. **New skill `plugins/reflect/skills/park/SKILL.md`.** Mirrors `catchup`'s five-question shape (goal · done · now · open · next) plus the git SHA at write time. `disable-model-invocation: true` (summoned only, same as every reflect member) — no context-percentage auto-trigger; a statusline showing context usage is a legitimate *cue* for a human to decide to park, never park's own trigger.
2. **✅ Where it writes (decided 2026-07-13 by Paul via `/shape:elicit`, before this ADR)**: a single file, **`HANDOFF.md` at the working project's root**, overwritten in place every time — not a dated chain, not buried in `.claude/`. Reasoning: the root is where `catchup`'s own Step 1 already looks first (an `ls` away); a dated chain leaves stale cursors lying around as **Sediment** (only the newest cursor is ever meaningful — history has its own owners: git for what shipped, `observe` for what was learned); and the A/B experiment's C arm — the one that scored cleanly — used exactly this shape. The two alternatives considered and rejected: `blueprints/plan.md`'s In-progress column (conflicts with `/shape:align`'s single-writer convention over that file, and not every project has a `blueprints/` tree) and a commit-message body (unwritable at the most common moment a session needs to stop — before there's anything committable). Whether to `git add`/commit `HANDOFF.md` is left to the user (local-only cursor by default; commit only for cross-machine work).
3. **`catchup` gains a consumption-priority step**: check for `HANDOFF.md` first; if its recorded SHA matches `git rev-parse HEAD`, read it as the primary source; on a mismatch, downgrade it to "possibly stale" and revalidate against fresh git state rather than trusting it outright. `catchup` reports which tier it read from.
4. **Content addition drawn from the A/B test, not the original proposal**: park's guidance explicitly instructs naming rejected/abandoned paths (not just what's next) — the single highest-value content the experiment isolated, and the one place a plain "next steps" framing (silence about a dead end implies it's still live) produces a confident wrong answer.
5. **✅ Family-fit (decided 2026-07-13, after Paul pushed back and the question was worked through before scaffolding)**: `reflect`'s membership test is *object* (does it face the work container?), not *tense* (past vs. forward) — the same test that already let forward-looking `catchup` ("now + next") into a family named for reflexivity rather than retrospection ([ADR-056](docs/adr/056-manage-renamed-reflect.md)). `park`'s object is squarely the work container (what I did, where I am, why) — it clears the same bar. The reversal-line, kept honest rather than declared settled forever: if reflect later accretes more session-management verbs (resume / pause / checkpoint), that's a *different* family (continuity) forming, and the tell is "the day every new member needs its own justification for fitting the name is the day the name stops fitting." Full walk-through: `blueprints/thoughts/2026-07-13-reflect-park-writes-the-handoff.md`.
6. **reflect moves from one writer to two.** `plugins/reflect/CLAUDE.md`'s "read-only by default, one writer" convention is rewritten: `catchup`/`park` are the read/write pair over the **cursor** (where + why); `summarize`/`observe` are the read/write pair over the **knowledge** (what happened + what's worth keeping). Two pairs, not a broken 3-to-1 asymmetry.

## Why

- Grounding a claim in a controlled test (not just three citations) matters here specifically because the failure mode isn't "less helpful" — it's **actively wrong with high confidence** (the bare-git arm's confabulated why). A gap that produces confident fabrication is a stronger case for a fix than a gap that merely produces a shrug.
- The unstructured-notes arm scoring 19/20 (equal to the structured arm on the raw rubric) matters for scope discipline: it means the *presence* of any written cursor buys most of the value, and the marginal value of `park`'s structure is concentrated specifically in **eliminating the false-positive dead-end trap** and **removing all hallucination**, not in raw information content. That's the honest justification for building a whole skill rather than telling users to jot a note: the delta is small in volume, load-bearing in kind.
- Root-level, single-file, overwrite-in-place was chosen over a dated chain because a *history* of cursors has no consumer — nothing reads yesterday's `HANDOFF.md` once today's exists, so keeping it around is the definition of Sediment (see `plugins/nav/skills/compose/references/authoring-failure-modes.md`, landed the same day by [ADR-069](docs/adr/069-adopt-prose-failure-vocabulary-plus-the-tell-column.md)) — it just happens to be a newly-introduced pile rather than an old one.
- The family-fit worry was worth resolving explicitly rather than assuming: a family whose membership test has to be re-argued every time a new verb is proposed is a family whose name is already under strain, so writing the resolution down (rather than letting it be implicit) is what lets a future proposal reuse the same test instead of re-litigating reflexivity from scratch.

## Consequences

- New: `plugins/reflect/skills/park/SKILL.md`.
- `plugins/reflect/skills/catchup/SKILL.md`: new consumption-priority step (check `HANDOFF.md` + SHA before falling back to plan artifacts, then pure git); companion-skills list gains `park`.
- `plugins/reflect/CLAUDE.md`: roster description rewritten as two read/write pairs; "one writer" convention rewritten to "two writers"; value-guardrail section gains `park`'s forced-structure justification and the family-fit check; `retro` renumbered from "planned 4th member" to "planned 5th."
- `plugins/shape/skills/build/SKILL.md` and `plugins/nav/skills/plan/SKILL.md` each gain one guarded, offer-only mention of `/reflect:park` at their natural stepping-away point — never automatic, and skippable when the harness already provides its own handoff/compaction mechanism.
- reflect's plugin version bumps 0.3.0 → 0.4.0 (new skill); registered in `README.md` and `docs/site/index.html` per the repo-root `CLAUDE.md`'s gate #3.
- `blueprints/thoughts/2026-07-13-reflect-park-writes-the-handoff.md` stays as the canonical write-up of the family-fit walk-through and the two ✅ decisions — this ADR references it rather than re-deriving it.
- **Not decided here**: whether `HANDOFF.md` should ever be committed by default for cross-machine work (left as a per-user call), and whether the dead-ends content should eventually become its own named field rather than folded into `Open`/`Done` — both are explicitly open for a later revision if real usage shows the fold-in reads awkwardly.
