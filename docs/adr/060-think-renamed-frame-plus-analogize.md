# ADR 060 ‚Äî `think` renamed to `frame`, plus a new member `analogize`

**Status**: accepted ‚Äî implemented 2026-07-03 (frame `0.5.0`)
**Date**: 2026-07-03
**Supersedes**: refines [ADR-034](docs/adr/034-think-plugin.md) (the plugin's birth + charter ‚Äî name and scope only; the four existing lenses are unchanged)

## Context

The user wanted a new, deliberately-invoked skill for building a well-crafted analogy ‚Äî not the ambient, reflexive plain-language-with-a-metaphor habit already codified as a standing style default, but a heavier, *summoned* version for a concept that isn't landing: compare multiple candidate analogies, check the mapping against the real thing, and say where the winner breaks down. The natural first instinct was to add it to `think`. On inspection, `think`'s own name and charter (ADR-034: "reasoning about a problem through a named lens") only ever fit one direction of transfer ‚Äî the agent deriving insight about a problem for *itself*. The new skill runs the opposite direction: the agent already holds the answer and needs to deliver it *to the user*. Cramming it into `think` would either misdescribe `think` (the name promises reasoning, not explaining) or force the new skill to pretend it's a reasoning lens when its whole point is delivery, not derivation.

The naming search (with the user, in-session) considered `understand`, `grasp`, `clarify`, and `frame` for the plugin; `translate`, `bridge`, `unpack`, `elucidate`, `illuminate`, and `analogize` for the new skill. `frame` won because it's literally accurate for *all five* members, not just a nice-sounding umbrella: the four existing lenses each apply a structural frame to a **problem** (first-principles strips it to axiomatic frame, dialectic applies a trial frame, orthogonal applies an independence frame, graft applies a donor-model frame); the new member applies a frame to an **answer** (an analogy is, literally, a cognitive frame ‚Äî mapping the unfamiliar onto a familiar structure). `analogize` won over its alternatives because it names the specific mechanism (build-and-check an analogy) rather than a generic outcome (`translate`, `explain` ‚Äî both fires on "say it differently" without committing to *how*).

## Decision

| Part | Decision |
|---|---|
| **Rename** | plugin `think` ‚Üí **`frame`**. The four existing lenses are unchanged in name, behavior, and output shape: `frame:first-principles` ¬∑ `frame:orthogonal` ¬∑ `frame:dialectic` ¬∑ `frame:graft`. |
| **New member** | **`frame:analogize`** ‚Äî build a deliberately stress-tested analogy for a concept the agent already understands. Generate 2‚Äì3 candidates from different source domains, check the mapping point-by-point against the target's real structure, pick on fit (not on which came first), and explicitly name where the winner still breaks down. Output: one plain sentence ‚Üí the analogy spelled out ‚Üí precise/technical detail layered after, only as needed ‚Üí where it breaks. |
| **Version** | `0.4.0 ‚Üí 0.5.0` (the version owner: `plugins/frame/.claude-plugin/plugin.json`) ‚Äî one bump covers both the rename and the new member, landed in the same commit. |
| **Positioning** (the identity the rename ratifies) | `frame` = apply an explicit, named frame that forces a structure the default skips, in either of two directions on the same object: **to a problem** (the four reasoning lenses, for the agent's own understanding) or **to an answer already in hand** (`analogize`, for the user's understanding). Same discipline, opposite direction of transfer. |
| **Naming idiom ‚Äî two, side by side** | The four lenses keep the canonical-lens-name divergence from ADR-027 (`first-principles`, not a coerced verb). `analogize` uses the marketplace's default bare-verb convention instead ‚Äî it names an action, not a citable reasoning concept. Both idioms are documented as deliberate in `plugins/frame/CLAUDE.md`; this is not drift. |
| **`analogize`'s trigger phrasing is provisional** | Ships as a complete, working `SKILL.md` (not a stub), but the `description` frontmatter's trigger-phrase list is a first cut. Per the user's explicit direction, refinement comes from real usage ‚Äî which phrasings actually preceded wanting it ‚Äî not upfront speculation. Future edits to sharpen it are normal skill maintenance, not a redesign. |

## Why

- **Direction-of-transfer is the axis that actually varies, and `think` only ever named one side of it.** The four lenses are *generative*: the agent derives an answer from the problem's own structure. `analogize` is *pedagogic*: the agent already holds the answer and delivers it into the user. `think` (reasoning about a problem) fits the first perfectly and the second not at all ‚Äî the fit-test that motivated ADR-056's `manage ‚Üí reflect` rename applies again here: a family name must describe every member on the shelf, not just the majority.
- **`frame` is accurate, not just evocative.** It isn't a metaphor bolted onto an unrelated skill set ‚Äî a reasoning lens and an analogy are both, literally, frames: a structure you deliberately select and apply to make something (a problem, an answer) tractable. The word does real descriptive work for both directions, which is why it survived a live differentiation check against the other five plugins (`nav`=navigability, `shape`=build, `research`=read, `reflect`=mirror, `relay`=coordinate ‚Äî none share this axis) and a real concern raised mid-search (does `frame` collide with `shape`'s "give intent its form"? Answer: no ‚Äî `shape` converges a decision into a disposable instance and builds it into running code; `frame` is purely in-chat, produces nothing to build, and the words diverge enough once the description text carries the distinction).
- **A single-skill addition didn't justify keeping the old name and accepting the mis-scope.** The alternative ‚Äî force `analogize` into `think` unmodified, description be damned ‚Äî was rejected because it repeats the exact failure ADR-056 already named and fixed once (`manage` mis-coloured its family; here `think` would mis-colour half of it).
- **Provisional-but-shipped, not speculative-and-parked.** Unlike the dropped `invert`/`second-order` seed (ADR-046, no usage evidence, never built), `analogize` ships as a real working skill because the user's need is already established (a documented, frequent, existing behavior they're currently getting from an ambient global-CLAUDE.md instruction that under-performs). What's provisional is only the exact trigger wording, which is properly discovered through use, not invented in one sitting.

## Consequences

- **Rename touches ~12 files across owner / generated / human-surface / cross-ref layers**, the same shape as ADR-056's impact list: `plugins/think/` ‚Üí `plugins/frame/` (git mv, preserves history); the four lens `SKILL.md` files' internal `/think:*` cross-references; `plugin.json` (name, version, description ‚Äî hand-owned); `.cursor-plugin/plugin.json` + `marketplace.json`'s `version` field (regenerated via `build-manifests.mjs`); `marketplace.json`'s `name`/`source`/`description` (hand-edited ‚Äî not derived); repo-root `CLAUDE.md` (one naming-convention example); `plugins/shape/CLAUDE.md`, `plugins/shape/skills/position/SKILL.md`, `plugins/shape/skills/position/references/write-protocol.md` (live `/think:first-principles` cross-refs); `README.md` (table row, cross-plugin sentence, skills section, three install-command surfaces); `docs/site/index.html` (VERIFIED list, `DOMAINS.frame`, the graph node ‚Äî gate #3 human surfaces); the Codex mirror + `AGENTS.md` (regenerated via `build-codex.mjs`, flat names `think-*` ‚Üí `frame-*` plus the new `frame-analogize`).
- **History is not rewritten; live cross-refs are.** Dated records ‚Äî ADRs 034/045/046/047/048/057, the `docs/observations/` entries, `docs/design/relay.md`'s dated changelog line ‚Äî keep `think:*` untouched; they were accurate when written. Only *live* descriptions of the *current* system update.
- **One green validator, staged commit.** Regenerate, run `node scripts/validate-codex-skills.mjs` to green (including the version-in-site-map gate added in the immediately preceding revision, ADR-driven by the same session's `/nav:audit` pass), commit the rename + new skill as one coherent change.
- **A marketplace rename is NOT an install migration (repeats ADR-056's caution).** Anyone with `think` installed must `claude plugin uninstall think@skills && claude plugin install frame@skills` ‚Äî the catalog rename alone leaves a dangling pinned install pointing at a name no longer in the catalog.
- **Opportunistic fix, same session:** `marketplace.json`'s hand-owned `relay` description was independently stale (still said "reply", missing `format`) ‚Äî corrected alongside the `think` entry while the file was already open for the rename, same class of drift a preceding `/nav:audit` pass had just found and fixed in `docs/site/index.html`.

## Out of scope

- **Sharpening `analogize`'s trigger phrasing** ‚Äî deliberately deferred to real usage, per the user's explicit direction. Not a gap; a design choice.
- **Renaming or re-scoping the four existing lenses** ‚Äî ADR-034's charter and ADR-027's canonical-lens-naming divergence stand unchanged; this ADR changes only the plugin's name and adds one new member.
- **Deciding whether future frame-on-an-answer members exist** ‚Äî `analogize` is evidence-gated like every other member (ADR-018); no further "delivery-direction" members are pre-listed here.
