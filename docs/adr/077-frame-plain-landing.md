# ADR 077 — frame's plain landing: a mandatory zero-jargon final step for all four reasoning lenses

**Status**: accepted
**Date**: 2026-07-14
**Source**: `blueprints/thoughts/2026-07-14-frame-plain-landing.md` (the ratified design)
**Extends**: none directly, but reuses the borrow-by-protocol pattern established by ADR-008 (`shape:position` borrows elicit/mockup/first-principles without invoking them) and the knowledge≠cue lesson documented live the same day in `docs/findings/2026-07-14-elicit-ecosystem-probes.md`.

## Context

`frame`'s four reasoning lenses — `first-principles`, `orthogonal`, `dialectic`, `graft` — each force a structure the default reasoning skips (assumptions→axioms→rebuild→divergence; tangle→axes→independence→payload; claim→steelman→attack→verdict; target→donor→map→adapt). That structure is the value, and it is optimized for **reasoning-verifiability**: a careful reader can check every step. It is not optimized for landing. Every lens's final output is the structured artifact itself, written in the lens's own terms of art — "the axiom is X, so the rebuilt conclusion is Y", "verdict: unsettled — owned bet", "the adapt list is the payload." The conclusion is visible, but it is not legible to someone who didn't hold the scaffolding in their head while reading it.

Paul (2026-07-14) ratified this as a **B-type pain**, distinct from A-type: the conclusion is not buried in the skeleton (A) — it's visible, but delivered in jargon the reader can't parse (B). The fix has to translate the last line, not reorder the skeleton.

### The evidence that a footer doesn't fire

Every lens already carries a "Communication Style" footer: "Always explain concepts using simple, direct, and plain language... Use analogies and metaphors frequently... Avoid academic jargon." All four lenses ship this footer today. All four lenses still produce jargon-terminal outputs — the footer is advisory prose sitting below the fold, and it does not change what the structured output above it says.

This is not a hypothetical failure mode; it is the same mode the elicit-ecosystem probes documented live, the same day, in a completely different skill. `docs/findings/2026-07-14-elicit-ecosystem-probes.md` ran a fresh cold-start agent against `shape:elicit`'s Gatekeeper section — a described diagnosis ("stop and offer `/shape:survey` when the user is missing a whole axis") sitting in a reference section of the SKILL.md. The agent read the section, understood it, and did not apply it at the moment it mattered — it kept grilling. Only after the diagnosis was rewired into the same **walked step** the volley loop already forces every turn (Protocol step 3, "stall tells — checked on the user's words every turn") did the behavior start to shift. The probes' own framing: *"a described diagnosis is available knowledge, not a forcing cue."* Frame's Communication Style footer is exactly this pattern — available knowledge, sitting in a section the model is not forced to re-consult at the moment of writing the final line. The remedy that worked for elicit (bind the check into the walked step the engine already executes every time) is the remedy this ADR applies to frame.

### The translation engine already lives in the family

`frame:analogize` is the member built exactly for this job: take an already-understood concept and land it — generate 2-3 candidate analogies, check the mapping structurally (not on vibes), pick on fit, name where the winner breaks. None of the other four lenses has ever used this discipline on their own output. The plugin's own through-line already states the two directions frame covers — lenses derive *for the agent*, analogize delivers *to the user* — and until now the derivation lenses never handed their result to the delivery discipline, even informally.

## Decision

1. **Each lens's protocol gains one final walked step: "Land it in plain words."** Not a rewrite of the existing structure, not a reordering — an addition appended after the lens's own last step (first-principles' and graft's numbered "The walk" gains step 6; dialectic's five-part trial table gains a sixth row; orthogonal's five-part output list, which doubles as its walk, gains a sixth bullet). The scaffolding above stays exactly as before, for anyone who wants to verify the reasoning — this step only adds a translation layer on top of it.
2. **The step's content is fixed across all four lenses**: one conclusion sentence with zero jargon, plus one analogy. The analogy is not invented ad hoc — it is chosen by borrowing `frame:analogize`'s own discipline **by protocol** (compare candidates, pick on fit, and — where checkable — name in half a sentence where the winner breaks), the same borrow-without-calling pattern ADR-008 established for `shape:position` reusing elicit/mockup/first-principles. `analogize` itself is never invoked — skills in this marketplace never call each other (ADR-007/015) — its discipline is restated inline, briefly, at the point of use.
3. **The zero-jargon rule is made mechanical, not aspirational, via a per-lens banned-term list** — the same fix pattern as elicit's stall-tells: a walked, checkable criterion beats a described intention every time a fresh agent runs cold.
   - `first-principles` bans: "axiom", "公理", "first principles"
   - `orthogonal` bans: "orthogonal", "正交", "axis" / "軸"
   - `dialectic` bans: "steelman", "dialectic"
   - `graft` bans: "graft", "primitive", "donor"

   These are the lens's own terms of art — the words load-bearing for verifying the *analysis*, and exactly the words a reader outside the analysis can't parse. Everyday paraphrases are the intended replacement ("the one thing this really rests on" for axiom, "these move independently" for orthogonal, "the strongest honest case for each side" for steelman, "borrowed structure that had to change shape" for adapt/graft) — the ban forces the paraphrase into existence rather than trusting the model to reach for one unprompted.
4. **Each lens gains one anti-pattern row**: ending on a jargon sentence is named explicitly as a refusal case — "that's analysis without landing." This closes the same gap the ADR's evidence section names: without a checkable anti-pattern, "be plain" reads as a style preference that competes with (and loses to) the model's default instinct to hand back the artifact it just built, verbatim.
5. **`frame:analogize` itself is untouched.** It is already the landing skill in full — nothing about this change asks it to do more, and editing it would be the N+1 mistake this repo's borrow-by-protocol pattern exists to avoid (ADR-008). Its Communication Style footer stays as-is; it was never the failure point — the four *lenses*' lack of a walked landing step was.
6. **All four lenses ship in one commit, not a pilot.** The four additions are structurally identical (one appended step, one banned-term list, one anti-pattern row), touch none of the four lenses' engines, and the criterion (does the final paragraph contain a banned term? does it contain an analogy?) is mechanically checkable in a single probe round across all four. A pilot's information gain here is low: the risk being tested (does a walked step actually fire, the way elicit's stall-tells needed three rounds to partially confirm) is the same risk for all four lenses, and a single-lens pilot would only re-derive what the elicit probes already established about the walked-step-vs-footer distinction, not learn something lens-specific. This is Paul's ratified call, not a default — recorded here as the reasoning, not asserted without one.

## Why

- **The footer's failure is not a frame-specific fluke — it's the same knowledge≠cue mode this repo has now observed twice in one day, in two unrelated skills.** Citing the elicit probes here is not decoration; it is the empirical basis for choosing "walked step" over "stronger wording in the existing footer" as the fix. The footer already asks nicely, four times over (once per lens) — the fix is not asking harder, it's moving the ask into the one place the engine is forced to pass through.
- **Borrowing analogize by protocol, not by call, keeps the family's invocation law intact and avoids a second landing engine.** A lens that invoked `/frame:analogize` mid-reasoning would violate the standing "skills never call each other" rule and would also be architecturally strange — analogize is scoped to explaining a settled concept to a user in a standalone turn, not to producing one paragraph inside another skill's output. Restating its discipline (compare, check structure, pick on fit, name the break) is cheap and keeps analogize's own file the single owner of that discipline's full definition.
- **A mechanical banned-term list is the only version of "no jargon" a cold agent can self-check.** "Plain language" is unfalsifiable in the moment of generation; "does this sentence contain the string 'axiom'" is not. This mirrors the general lesson from the elicit probes: a criterion has to be checkable to survive contact with a model that is not actively trying to violate it, just not actively forced to remember it either.
- **Analogize stays untouched because it was never broken — the gap was upstream of it.** The four lenses simply never fed their conclusion through any translation discipline at all; the fix is giving them one, by reference, not modifying the one member that already had it right.
- **All-four-at-once is justified by low marginal information, not urgency.** The four additions are mechanically identical in shape; running one lens as a pilot would test the walked-step-vs-footer distinction once and then require three more near-identical edits anyway. Given the engine of each lens is untouched (only an appended step + a banned-term list + one table row), the blast radius of being wrong is one probe round to catch it across all four, not a cascading rewrite.

## Scope note

This is an **in-chat output-shape change only.** No lens's frontmatter `description` changes, so no routing behavior changes — a lens still fires on exactly the same trigger phrasings as before. No lens's forced structure (the five-part output that IS the value, per each lens's own Core section) is reordered or shortened; the walked landing step is additive, appended after the existing structure, every time. `analogize`'s SKILL.md has zero byte changes.
