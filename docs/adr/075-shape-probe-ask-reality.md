# ADR 075 — `shape:probe`: ask reality when nobody knows

**Status**: accepted
**Date**: 2026-07-14
**Source**: `blueprints/thoughts/2026-07-14-elicit-ecosystem-four-quadrants.md` (the four-quadrant design rulings) · `blueprints/plans/2026-07-14-elicit-ecosystem.md` (Phase 2 of the implementation plan)
**Phase**: 2 of 4 (`survey` → `probe` → elicit gatekeeper upgrade → behavioral testing); this ADR covers `probe` and the `frame:dialectic` bridge only.
**Extends**: [ADR-074](074-shape-survey-decision-space-mapping.md) (Phase 1, `survey`) · [ADR-062](062-live-llm-cost-signal.md) (live-LLM-cost signal, `probe` gets its own instance) · [ADR-067](067-dispatch-tiers-consultant-seat.md) (dispatch tiers, design/verdict stay strong-seat) · [ADR-012](012-nav-plan-offers-visual-summary.md) (guarded cross-plugin offer pattern, reused for the dialectic→probe bridge) · [ADR-071](071-contracts-vs-conventions-tolerant-reader.md) (tolerant reader for probe's output doc).

## Context

The four-quadrant analysis (`blueprints/thoughts/2026-07-14-elicit-ecosystem-four-quadrants.md`) names a knowledge-state map wider than `/shape:elicit` alone serves:

| Knowledge state | Solved by | Verb |
|---|---|---|
| I know I know | Draw it out and align | `shape:elicit` (existing) |
| I know I don't know (the world knows) | Map → fill the gap → re-align | `shape:survey` (ADR-074, Phase 1) |
| I don't know I don't know (the world knows) | elicit detects it mid-grill → offers survey | elicit's gatekeeper role (Phase 3) |
| **Nobody knows** | Design a minimal experiment, let reality answer | **`shape:probe` (this ADR)** |

`elicit` and `survey` both assume an answer exists somewhere retrievable — in the user's head, or in the world's documented knowledge. The fourth quadrant breaks that assumption entirely: the fact hasn't been measured yet, by anyone. Grilling the user produces nothing (there's nothing to draw out); mapping the decision space produces nothing (there's no existing knowledge to map against). Arguing longer just produces more confident-sounding noise. The only way out is to design a minimal experiment and let a measured result — not the strongest rhetoric — decide.

**Trigger evidence — this repo hand-rolled the deliverable at least three times:**

1. **The `/reflect:park` A/B comparison** (`docs/findings/2026-07-13-park-ab-experiment.md`) — three arms (bare git, ad-hoc notes, a structured handoff doc) tested against synthetic mid-refactor repos with a planted red-herring, six mutually-blind Sonnet subjects, one blind judge scoring against an answer key. Hand-designed from scratch: arm structure, red-herring control, blind judging, a scoring rubric.
2. **The 51-phrase blind routing test** (`docs/findings/2026-07-13-description-diet-rollout.md`, "Follow-up: blind old-vs-new routing experiment") — two roster documents (pre/post description-diet), 51 test phrases (35 direct + 16 adversarial confusable-pair phrases), two independent fresh Sonnet judges each shown only one roster, neither told the expected answer or that this was an A/B test. Hand-designed: phrase set construction, judge blinding, the explicit "neither told this is an A/B test" control.
3. **The tolerant-reader three-state behavior probes** (commit `b4c9a14`, "tolerant-reader behavior probes 3/3 pass") — a messy sandbox repo (code + an ad-hoc notes file with a planted dead-end, zero canonical structure) driven against three skills (`reconcile`, `build`, `catchup`) to observe live behavior against ADR-071's three-state tolerant-reader contract, rather than reasoning about whether the contract held.

Three independent instances, each re-inventing the same discipline (arm/variant design, a control, blinding where relevant, a scoring or verdict rule) from scratch, with no shared verb naming it. This is ADR-038's trigger condition verbatim: **a verb was missing.** The `SKILL.md`'s three canonical experiment shapes — A/B comparison, blind judgment test, behavior probe — are these three instances genericized and stripped of this repo's own domain nouns, not invented in the abstract.

**The broken hand-off `probe` closes:** `frame:dialectic`'s five-part trial structure ends with a "Killer Experiment" row — the one experiment that would decide the claim's fate — but dialectic itself never runs it (frame writes no file, decides nothing, builds nothing; its Companion skills route to `/shape:elicit`, `/shape:mockup`, `/nav:plan`, none of which actually execute an experiment). Every dialectic trial that lands on "unsettled — owned bet" names a decider and then has nowhere to send it. `probe` is that destination.

## Decision

1. **`probe` is a new `shape` skill whose identity is "ask reality," not "reason harder."** It fires only when a fork is genuinely undecidable by argument — not merely under-argued. If more reasoning, more grilling, or more mapping could still move the answer, that's `elicit` or `survey`'s territory, not this one.
2. **The design chain borrows three frame lenses by protocol, not by call**, in sequence, each credited: (a) `frame:dialectic`'s move — locate the single load-bearing assumption the fork actually rests on (reusing its "Missing Evidence" row directly when a dialectic trial already ran); (b) `frame:first-principles`' move — strip that assumption to the smallest falsifiable claim, testing necessity rather than the surface phrasing it was originally argued in; (c) `frame:orthogonal`'s move — control the variables, varying exactly the one axis the stripped claim is about and locking every other axis identical. Orthogonal's independence check (move one axis, the others must not move) **is, without translation, the definition of a controlled experiment** — this is not an analogy, it's the same check applied to a live trial instead of a conceptual factoring. This mirrors the borrow-by-protocol pattern `shape:position` established for elicit/mockup/first-principles (ADR-008) and `shape:survey` established for `frame:orthogonal` alone (ADR-074) — `probe` is the first skill in this family to chain three borrowed lenses in sequence rather than one.
3. **Three canonical, generic experiment shapes**, each stack-neutral and standalone-legible (no origin-project nouns): an A/B comparison of two variants under identical conditions; a blind judgment test with independent judges neither told the expected answer; a behavior probe that drives the real system in a controlled state and observes rather than reasons. These are the three trigger-evidence instances above, genericized — canonical because they recur, not because they're exhaustive; a fourth shape earns its place by the same bar (recurring real use), not by invention per-session.
4. **Pre-registration is the non-negotiable core.** The verdict rule — what result maps to which conclusion — must be written down before the first trial runs. An experiment whose "what counts as a win" gets decided after the data arrives isn't an experiment; it's a rationalization wearing an experiment's clothes. `probe` refuses to run without this.
5. **Dispatch splits at the same line ADR-067 already draws**: design (the three-lens chain) and verdict-reading against the pre-registered rule are judgment-dense and stay with the session model; the execution legs — running each variant, collecting raw outputs — can go to cheap-tier delegated agents, reporting grounded in fact (the actual output, the actual number), never an impression. ("Delegated agent," not "sub-agent" — the Codex compat ratchet denylists the literal token `sub-agent` in generated mirror files; the phrasing in `probe`'s `SKILL.md` and this ADR uses "delegated agent" throughout for that reason.)
6. **A new, `probe`-specific instance of the live-LLM-cost signal (ADR-062).** ADR-062's existing instances all sit inside `nav:do`'s inherited verify-gate chain or shape's own dogfood/build/setup real-app-driving steps — none of them cover an experiment that itself fans out paid-LLM calls (multiple judges, N repeated trials). `probe` states the expected scale (model, call count, trial count) and gets a nod from the user before running, the same aware-while-writing/notify-at-the-gate posture, never an automatic switch to a cheaper path.
7. **Read-only toward product code; the harness is disposable, the findings doc is the residue.** `probe` never edits source to make an experiment work — a change to product code changes what's under test, which isn't an experiment anymore. Fixtures, scripts, and transcripts produced while running one are scratch/throwaway; only the findings doc (method + data + verdict) lands durably.
8. **Output lands `docs/findings/` when that convention exists in the project, else `blueprints/thoughts/<date>-<topic>-probe.md`**, tolerant-reader compliant (ADR-071): scaffold the simpler of the two if neither exists, tolerate a non-standard shape, self-report which tier was written to. States explicitly that the evidence feeds the pending decision back — into elicit's next volley, or the user directly.
9. **`frame:dialectic`'s closing offer gains a guarded `/shape:probe` option**, worded like `nav:plan`'s existing guarded `/shape:mockup` offer (ADR-012 pattern): offered, never auto-called, and it degrades gracefully — omitted, not broken — if `shape` isn't installed. This is a **frame → shape soft offer**, the same direction every other frame lens already uses to hand off an in-chat analysis to a persisting shape verb; it adds no new dependency because frame's Companion-skill offers were already this shape (guarded, one-shot, degrade-if-absent).

## Why

- **"Nobody knows" is a real, distinct knowledge state, not a rebranding of "hasn't been argued enough."** The four-quadrant thought treats it as structurally different from the other three — the other three all have an answer *somewhere retrievable*; this one doesn't, until measured. A skill that tried to serve this state by reasoning harder (a longer dialectic trial, a deeper survey) would just produce more elaborate noise; the fix has to be a different kind of move (measurement), not more of the same kind (argument).
- **The three-lens design chain is the disciplined middle between "just try something" and "reason it out fully."** Without step (a), an experiment tests whatever axis happened to come up in conversation, not the one that actually decides the fork. Without step (b), the experiment tests a vague or compound claim that can't cleanly falsify. Without step (c), two things vary at once and the result can't be attributed to either — polluted evidence, worse than no evidence, because it looks like an answer.
- **The trigger evidence is real, not constructed for the ADR.** All three hand-rolled instances predate this decision by at most a day and were built independently of each other and of any plan to eventually generalize them — exactly the kind of unplanned recurrence ADR-038 treats as the signal a verb is missing, as opposed to a single one-off that doesn't yet justify a dedicated skill.
- **Closing the dialectic hand-off matters because the gap was silent, not loud.** A trial that lands "unsettled — owned bet" already tells the user they're building on an unproven assumption; without `probe`, the "Killer Experiment" row was permanently aspirational — named, never actually runnable through any verb in the marketplace. `probe` makes that row a real next step instead of a stated-but-unreachable good intention.
- **Reusing ADR-012's guarded-offer wording for the dialectic bridge** avoids inventing a new cross-plugin dependency shape when one already exists and is proven: `nav:plan`'s guarded `/shape:mockup` offer already established "degrade gracefully if the target plugin isn't installed" as this marketplace's answer to a soft, one-way, guarded cross-plugin hand-off — frame's lenses already offer into shape this way for `elicit`/`mockup`/`plan`; adding `probe` to that list is not a new pattern, just a new target.

## Overlap / boundary analysis — four neighbors

- **vs `/shape:mockup`** — the sharpest boundary, because both sit on the shape spine "converge by a real, disposable instance, never a description." `mockup` is that spine's face for **preference-decidable** questions: render candidates, the user looks and picks. `probe` is the same spine's face for **fact-decidable** questions: run candidates under controlled conditions, reality picks. Neither substitutes for the other — a look-and-feel question doesn't get settled by a controlled experiment, and a factual claim about system behavior doesn't get settled by which option a person prefers to look at.
- **vs `/shape:dogfood`** — dogfood drives an **already-built** feature to surface friction and coverage gaps in something real that exists; its object is "how does this feel to use." `probe` answers a **still-undecided** question, often before either candidate is fully built — the experiment itself may be the first artifact either variant gets. Different object entirely (a shipped feature's UX vs an open factual unknown), so trigger overlap is low.
- **vs `/verify`** — `verify` checks that a **change did what it claims** — a known, stated expectation, confirmed against the running system. `probe` answers an **open unknown** where no expectation has been stated yet, because nobody knows what's true. `verify` confirms; `probe` discovers.
- **vs `/research:critique`** — critique audits an **external paper's existing evidence** (validated ≠ claimed, is the evidence in the paper strong enough for what it claims). `probe` generates **new evidence for your own open question** — there is no existing paper or document to audit; the evidence doesn't exist yet and `probe`'s job is to produce it. One is forensic (auditing evidence that's there); the other is generative (designing evidence that isn't).

Trigger-collision risk overall is low because each neighbor's core question differs from probe's own ("is this the look I want" / "does this built thing feel smooth" / "did my change do what I said" / "is this paper's existing evidence strong enough" — none of these is "nobody knows the answer to X, how do we find out"). The `SKILL.md` frontmatter's boundary sentences restate these four lines so a routing model sees them without opening the body.
