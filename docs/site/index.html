Skills — marketplace map

Personal Claude marketplace. A growing collection of focused, single-purpose plugins for keeping code navigable as it grows. Three layers: a marketplace (the container) holds plugins (topical families) which hold skills (the verbs you actually invoke).

How to read this: scroll for a 30-second overview, or jump to any section from the left. Every domain graph is interactive — click a node for its details, drag to rearrange.

Built on Ousterhout's deep-module principles. Self-contained skills (no shared imports). Language-agnostic (universal core + per-stack heuristics).

Install — two commands

Add the marketplace, install the nav plugin. All 6 skills become available as /nav:<verb>.

/plugin marketplace add ChenPaulYu/skills
/plugin install nav@skills

Iterating on this marketplace itself? Use a local absolute path instead of the GitHub handle — see README → Local development.

Architecture — three layers, one container

The marketplace registers via .claude-plugin/marketplace.json and lists which plugins live inside. Each plugin declares itself via .claude-plugin/plugin.json and exposes skills under skills/<verb>/SKILL.md. Slash commands surface as /<plugin>:<skill>.

plugin nav anatomy — assess, transform, describe, orchestrate

Six skills, one verb each. audit is the source of truth (read-only). Two meta-skills inline it: doctor sequences light fixes (audit → headers → map); plan grounds a spec then clarifies + writes a plan artifact. refactor stays standalone (heavy-touch, narrow scope). headers output feeds map input — separate by design (ADR-003).

Neither meta-skill auto-acts on what it surfaces. doctor recommends concrete /nav:refactor invocations; plan writes a plan artifact the user reads before executing. Both inline audit's protocol but follow the "scan transcript first" preamble — if audit ran this session, they reuse its output instead of re-running grep (ADR-006).
